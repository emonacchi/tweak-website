FAQs

Will this work in every web application?

No. tweak has its own limitations, currently:
  • Is not able to intercept requests that are triggered at the page load. The problem is that the requests might be fired before the extension loads in the web page. This is a race condition we are putting a lot of effort to solve.
  • It cannot intercept data received through WebSockets.
  • It will only apply the configurations to the browser tab where you configure the requests to intercept and activate the extension (by clicking the run button).
  • In case you refresh the page of the website while the extension is active the extension will need a manual reset, you'll need to click on the STOP button and then again click on the RUN button in order to the extension to be able to intercept requests again.

Is tweak interfering with other web applications?

tweak uses Window.postMessage API to communicate between web pages and the extension. Leaving tweak turned on (running mode) might interfere with the runtime of the website. It is a good practice to keep tweak paused and only turn it on when you are using it.

Another important aspect on the request URL matching expression that might be easy to miss, is that sometimes we perform requests to the same domain, thus we do not have the domain present in the request URL, we simply write the path. Will tweak ever be available for Mozilla Firefox?

Perhaps. There are no plans to do so at the moment, we might add it if there's a high number of requests.

Any thoughts on having a tailored support for GraphQL?

We're seriously considering having something in the extension very tailored to GraphQL to intercept request that interact with GraphQL servers. Send us an email to help.tweak@gmail.com expressing your interest.
